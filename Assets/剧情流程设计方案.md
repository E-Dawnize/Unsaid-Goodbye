# 剧情流程设计方案

> 游戏：饲主失踪之后 (Unsaid Goodbye) | 2026-04-30

---

## 1. 流程特征分析

| 特征 | 描述 |
|------|------|
| 整体结构 | 线性推进，仅结局处 1 个分支 |
| 场景数 | 7 个主场景（2表世界 + 3里世界 + 2后日谈） |
| 剧情节点 | 18 个 |
| 场景内探索 | 顺序自由，完成后触发场景转换 |
| 分支条件 | 道具收集数量决定结局 A/B |
| 复杂度 | 低 — 无网状分支、无多线程剧情、无角色好感度 |

---

## 2. 方案对比

### 方案 A：纯代码状态机

直接在 `SceneFlowController` 中用 `switch(phase)` 硬编码所有转换逻辑。

```
优点：零额外文件，最快实现
缺点：修改需改代码重编译，策划无法调整流程
适合：单人开发、极小项目
```

### 方案 B：ScriptableObject 阶段配置 + 事件驱动状态机（推荐）

每个阶段用 ScriptableObject 存储配置，`SceneFlowController` 读取配置 + 订阅事件驱动推进。

```
优点：策划可在 Editor 配置流程，代码简洁，存档友好，两人协作清晰
缺点：需创建 ScriptableObject 资产文件
适合：2-8人团队、需要策划迭代的中小型项目
```

### 方案 C：节点图系统

用 xNode / Unity GraphView 构建可视化节点编辑器。

```
优点：可视化、最灵活
缺点：开发成本高（2-4天仅搭建框架），线性流程用不上灵活性
适合：复杂分支叙事游戏（VA-11 Hall-A、Coffee Talk 级别）
```

### 方案 D：Unity Timeline + Signals

用 Timeline 资产编排每个场景的叙事序列。

```
优点：Unity 原生、可视化时间轴
缺点：与道具/解谜系统集成困难，Timeline 绑定场景不灵活
适合：固定过场动画、镜头序列
```

### 方案 E：行为树 (Behavior Tree)

```
优点：模块化、可复用
缺点：设计用于 AI 行为而非叙事流程
适合：复杂 NPC 行为控制
```

## 3. 推荐方案：B — ScriptableObject 阶段配置 + 事件驱动状态机

### 3.1 核心思路

```
┌─────────────────────────────────────────────────────┐
│                    宏观层（代码）                      │
│  GamePhase 枚举 → SceneFlowController 状态机         │
│  固定阶段顺序，阶段间转换逻辑由代码保证                │
├─────────────────────────────────────────────────────┤
│                    配置层（ScriptableObject）          │
│  GamePhaseConfig 定义每个阶段：                       │
│  场景 / 音乐 / StoryBeat列表 / 下一阶段               │
├─────────────────────────────────────────────────────┤
│                    通信层（EventManager）             │
│  道具收集 / 解谜完成 / 对话结束 → 事件 → 匹配Beat    │
└─────────────────────────────────────────────────────┘
```

### 3.2 推进流程示意

```
玩家拿起药瓶(H0)
  → PickupInteractable.OnInteract()
  → InventoryManager.CollectItem("H0")
  → EventCenter.Publish(ItemCollectedEvent { ItemId = "H0" })
  → SceneFlowController 收到事件
  → TryCompleteBeat(CollectItem, "H0")
  → beat "collect_H0" 标记完成 ✓
  → 检查: 所有 RequiredBeats 都完成？
  → 否 → 等待更多事件

玩家调查手机(H1) → ... → beat "collect_H1" 标记完成 ✓
  → 检查: 所有 RequiredBeats 都完成？
  → 是 → OnPhaseComplete()
       → 转场黑屏 → TransitionToPhase(Phase2_InnerPark)
```

### 3.3 场景内任意顺序探索 到 场景转换 的模型

每个阶段是一个"房间"：进入后可以在房间内自由探索（收集、调查、解谜），完成所有"必做事项"后门自动打开（阶段推进）。

```
Phase1_SurfaceLivingRoom_Investigate:
  RequiredBeats:
    ├── collect_H0  (药瓶)
    ├── collect_H1  (手机)
    └── collect_H2  (空相框)
  
  → 三个可以任意顺序完成
  → 全部完成后 → 猫碗触发 → Phase2_InnerPark
```

// Game/SceneFlow/StoryBeat.cs
namespace Game.SceneFlow
{
    public enum StoryBeatType
    {
        CollectItem,        // 收集道具 → ItemCollectedEvent
        SolvePuzzle,        // 完成解谜 → PuzzleSolvedEvent
        CompleteDialogue,   // 完成对话 → DialogueEndedEvent
        InteractWithObject, // 交互特定物体 → InteractionPerformedEvent
        EnterTrigger,       // 进入触发区域 → TriggerEnterEvent
    }

    [System.Serializable]
    public struct StoryBeat
    {
        public string BeatId;          // 唯一ID，如 "collect_H0"
        public StoryBeatType Type;      // 节拍类型
        public string TargetId;        // 目标ID（道具ID / 解谜ID / 对话ID / 交互物ID）

        [UnityEngine.TextArea(1, 2)]
        public string Description;     // 策划注释："收集客厅茶几上的药瓶H0"
    }
}

### 4.4 SceneFlowController（核心）

```csharp
// Game/SceneFlow/SceneFlowController.cs
namespace Game.SceneFlow
{
    public class SceneFlowController : IInitializable, IDisposable
    {
        [Inject] private IEventCenter _events;
        [Inject] private ISaveManager _save;
        [Inject] private IAudioManager _audio;
        [Inject] private IInventoryManager _inventory;
        [Inject] private IEndingDeterminer _ending;

        // 所有阶段的配置（从 Resources 加载）
        private Dictionary<GamePhase, GamePhaseConfig> _configs;
        private GamePhaseConfig _currentPhase;
        private HashSet<string> _completedBeats = new();
        private bool _isTransitioning;

        public GamePhase CurrentPhase => _currentPhase?.PhaseId ?? GamePhase.None;
        public event Action<GamePhase> OnPhaseChanged;
        public event Action<GamePhase> OnPhaseComplete;

        #region 初始化

        public void Initialize()
        {
            LoadAllConfigs();
            SubscribeEvents();

            if (_save.HasSave)
                RestoreFromSave();
            else
                StartPhase(GamePhase.Phase1_SurfaceLivingRoom_Initial);
        }

        private void LoadAllConfigs()
        {
            _configs = Resources.LoadAll<GamePhaseConfig>("Configs/Phases")
                .ToDictionary(c => c.PhaseId);
        }

        private void SubscribeEvents()
        {
            _events.Subscribe<ItemCollectedEvent>(e =>
                TryCompleteBeat(StoryBeatType.CollectItem, e.ItemId));

            _events.Subscribe<PuzzleSolvedEvent>(e =>
                TryCompleteBeat(StoryBeatType.SolvePuzzle, e.PuzzleId));

            _events.Subscribe<DialogueEndedEvent>(e =>
                TryCompleteBeat(StoryBeatType.CompleteDialogue, e.DialogueId));

            _events.Subscribe<InteractionPerformedEvent>(e =>
                TryCompleteBeat(StoryBeatType.InteractWithObject, e.InteractableId));
        }

        #endregion

        #region Beat 追踪

        private void TryCompleteBeat(StoryBeatType type, string targetId)
        {
            if (_isTransitioning || _currentPhase == null) return;

            foreach (var beat in _currentPhase.RequiredBeats)
            {
                if (beat.Type == type && beat.TargetId == targetId)
                {
                    if (_completedBeats.Add(beat.BeatId))
                    {
                        Debug.Log($"[SceneFlow] Beat 完成: {beat.BeatId} ({beat.Description})");
                        _events.Publish(new StoryBeatCompletedEvent { BeatId = beat.BeatId });
                    }
                }
            }

            if (AllBeatsDone())
                OnPhaseComplete();
        }

        private bool AllBeatsDone()
        {
            return _currentPhase.RequiredBeats.All(b => _completedBeats.Contains(b.BeatId));
        }

        #endregion

        #region 阶段推进

        private async void OnPhaseComplete()
        {
            _isTransitioning = true;
            OnPhaseComplete?.Invoke(_currentPhase.PhaseId);

            // 1. 播放离开对话
            if (!string.IsNullOrEmpty(_currentPhase.ExitDialogueId))
            {
                await PlayDialogueAndWait(_currentPhase.ExitDialogueId);
            }

            // 2. 确定下一阶段
            var nextPhase = DetermineNextPhase();

            // 3. 执行转场
            await TransitionToPhase(nextPhase);
        }

        private GamePhase DetermineNextPhase()
        {
            if (_currentPhase.IsEndingBranch)
            {
                var result = _ending.DetermineEnding(_inventory);
                return result == EndingType.B_Leave
                    ? GamePhase.Phase7_Epilogue_B
                    : GamePhase.Phase7_Epilogue_A;
            }

            return _currentPhase.DefaultNextPhase;
        }

        private async Task TransitionToPhase(GamePhase nextPhase)
        {
            if (!_configs.TryGetValue(nextPhase, out var config))
            {
                Debug.LogError($"[SceneFlow] 找不到阶段配置: {nextPhase}");
                return;
            }

            Debug.Log($"[SceneFlow] 阶段转换: {_currentPhase?.PhaseId} → {nextPhase}");

            // 1. 黑屏淡入
            await FadeToBlack(_currentPhase?.TransitionDuration ?? 2f);

            // 2. 加载场景
            await LoadScene(config.SceneAssetPath);

            // 3. 重置当前阶段状态
            _currentPhase = config;
            _completedBeats.Clear();

            // 4. 切换BGM
            _audio.PlayBGM(config.BackgroundMusic);

            // 5. 黑屏淡出
            await FadeFromBlack(config.TransitionDuration);

            // 6. 播放进入对话
            if (!string.IsNullOrEmpty(config.EntryDialogueId))
            {
                await PlayDialogueAndWait(config.EntryDialogueId);
            }

            _isTransitioning = false;
            OnPhaseChanged?.Invoke(nextPhase);

            // 7. 存档
            _save.Save();

            Debug.Log($"[SceneFlow] 进入阶段: {nextPhase} ({config.DisplayName})");
        }

        #endregion

        #region 存档

        public void RestoreFromSave()
        {
            var data = _save.Load();
            JumpToPhase(data.CurrentPhase, data.CompletedBeatIds);
        }

        public void JumpToPhase(GamePhase phase, List<string> completedBeats)
        {
            if (!_configs.TryGetValue(phase, out var config)) return;

            _currentPhase = config;
            _completedBeats = new HashSet<string>(completedBeats ?? new List<string>());

            // 加载场景但不播放转场动画（直接跳转）
            UnityEngine.SceneManagement.SceneManager.LoadScene(config.SceneAssetPath);
            _audio.PlayBGM(config.BackgroundMusic, crossfadeDuration: 0f);

            OnPhaseChanged?.Invoke(phase);
        }

        public GameSaveData GetSaveState()
        {
            return new GameSaveData
            {
                CurrentPhase = _currentPhase?.PhaseId ?? GamePhase.None,
                CompletedBeatIds = _completedBeats.ToList(),
            };
        }

        #endregion

        public void Dispose()
        {
            // 取消事件订阅（由 DI 容器管理生命周期时自动处理）
        }
    }
}
```

## 7. 关键设计决策

### 7.1 为什么 Beat 用 struct 而非 class / ScriptableObject

- `StoryBeat` 是轻量数据，生命周期依附于 `GamePhaseConfig`
- 不需要单独引用、不需要在 Inspector 中作为独立资产拖拽
- `[System.Serializable] struct` 可以在 Inspector 中内联编辑，简洁够用

### 7.2 为什么阶段转换用 async/await 而非 Coroutine

- 项目已经在用 `System.Threading.Tasks`（AsyncCommand 中）
- `async/await` 写顺序逻辑更直观
- `SceneFlowController` 不是 `MonoBehaviour`，不自然获得 Coroutine 能力
- 需要添加 `UniTask` 或自定义 `AsyncOperation` 包装以支持 `await SceneManager.LoadSceneAsync`

### 7.3 为什么转场和对话在 SceneFlowController 中处理而非独立系统

- 转场（黑屏）和阶段对话是流程控制的一部分，不是独立功能
- 放在 SceneFlowController 中避免了系统间复杂的回调/事件链
- `PlayDialogueAndWait()` 内部仍委托给 `IDialogueManager`，只是等待其完成事件

### 7.4 为什么不用 Unity 的 SceneManagement 事件驱动加载

- `SceneScopeRunner` 现有的 `sceneLoaded/sceneUnloaded` 机制保留给场景级 DI Scope 管理
- SceneFlowController 负责**流程层面**的场景调度（哪个阶段加载哪个场景）
- SceneScopeRunner 负责**容器层面**的场景 Scope 创建/销毁
- 两者职责不同，互不冲突

## 9. 与总体架构的关系

此模块属于 **服务层 (Services)** 中的「场景流程」系统，在总体架构中的位置：

```
表现层:      UI (对话框/选项框/交互提示)
              ↕ (通过 ViewModel 读取状态)
游戏逻辑层:  Interaction → Inventory → Puzzle → Narrative
              ↘ 全部通过 EventManager 发布事件 ↙
服务层:      ★ SceneFlowController (本模块)
              ↕
基础设施层:  DI / EventManager / Lifecycle / SaveManager
```

SceneFlowController 不直接依赖任何游戏逻辑层的具体实现，只通过 DI 注入接口 + 事件订阅完成协作。
